# Video Sourcing Contract

## 目标

Video Search Agent 用于为 AI 互动课件生成真实生活视频入口，减少教师手工搜索成本。当前策略是“最高相关推荐 + 候选链接 + 教师审核 + 网页跳转或平台嵌入”，不下载、不搬运、不二次托管第三方视频。

## 输入

- 课程主题。
- 应用任务 `missionId`。
- 搜索关键词。
- 教学适配标准。

输入配置位于：

```text
src/lib/videoAssets.ts
```

## 输出

默认输出：

```text
docs/reports/video_candidates.json
docs/reports/video_candidates.md
```

候选信息包括：

- 任务 ID
- 搜索词
- 最高相关推荐视频
- 备用候选链接
- 标题
- 作者/UP 主
- 原始链接
- 封面链接
- 时长
- 推荐理由
- 教师审核标准

## 禁止事项

- 不自动下载 B 站或其他平台视频。
- 不绕过平台播放、登录、风控或版权限制。
- 不把未授权视频复制到 `public/videos/`。
- 不把搜索结果直接标记为“已采用”，必须由教师确认。
- 不使用包含可识别学生人脸、车牌、学校名称或明显商业广告的视频。

## 推荐流程

```text
Video Search Agent 提供最高相关推荐
→ 学生/教师可直接跳转观看
→ 教师打开候选链接审核
→ 教师选择可用视频
→ 课件中使用跳转链接或平台允许的嵌入方式
→ 如有授权素材，再放入 public/videos/
```

## 合规说明

当前脚本只生成候选链接和报告，不下载视频内容。若课堂需要公开展示第三方视频，教师或发布方应确认平台规则、版权状态和展示场景是否允许。

## 命令

```bash
npm run search:videos
```
