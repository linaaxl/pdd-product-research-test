# 本地链接与教师共创访问契约

## 目标

开发环境需要同时服务三类对象：

- Codex/开发 Agent：需要稳定、短路径、适合热更新和自动化检查的链接。
- 项目负责人：需要本机打开即可预览，不被终端输出和技术细节干扰。
- 高中物理老师：需要一个可直接打开的共创评审链接，并清楚知道打不开时该找谁处理。

## 链接分级

| 等级 | 推荐对象 | 链接形式 | 风险 |
| --- | --- | --- | --- |
| 开发调试 | Codex/开发者 | `http://127.0.0.1:5173/` | 只能本机访问 |
| 本机预览 | 项目负责人 | `http://localhost:5173/` | 终端关闭后失效 |
| 局域网共创 | 高中物理老师 | `http://<开发机局域网IP>:5173/` | 受同网段和防火墙影响 |
| 发布评审 | 高中物理老师/教研组 | 学校服务器或云端 URL | 需要部署 |

## 推荐命令

```bash
npm run links
npm run dev:local
npm run dev:share
npm run build
npm run preview:share
```

## 交付给老师时的原则

- 不把 `127.0.0.1` 发给老师。
- 不要求老师安装 Node 或运行 npm。
- 给老师的是“评审链接 + 评审问题”，不是源码目录。
- 课堂正式使用前优先部署为稳定 Web 链接。
- 外部视频使用网页跳转和审核入口，不能把第三方视频默认打包成本地素材。

## 共创评审反馈格式

```text
课题：
评审老师：
使用设备：
是否能打开链接：
最能提高兴趣的场景：
最不贴近学生的场景：
AI 追问是否准确：
仿真操作是否顺手：
建议新增或删除的内容：
是否适合 40 分钟课堂：
```
