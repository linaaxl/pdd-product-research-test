# Harness Master Plan

## 项目目标

建立一个高中物理 AI 交互课件多 Agent 开发框架，并用第一节样板课《电磁感应：磁铁穿过线圈》验证该框架。

## 样板课范围

- 年级：高二物理
- 主题：电磁感应
- 核心问题：磁铁穿过线圈时，磁通量如何变化？感应电动势如何变化？感应电流方向如何由楞次定律判断？
- 核心交互：拖动磁铁穿过线圈，观察磁通量、感应电动势和电流方向变化。

## 阶段流程

1. 课程设计阶段
   - 产出课程目标、学情分析、问题链、课堂流程。
   - 负责人：Curriculum Agent。

2. 交互 Spec 阶段
   - 产出交互状态、控件、数据流、多表征展示规范。
   - 负责人：Interaction Design Agent。

3. 物理模型阶段
   - 产出 pure functions、边界条件、物理模型测试。
   - 负责人：Simulation Agent。

4. React 交互实现阶段
   - 产出可运行页面、SVG 仿真、曲线和控件。
   - 负责人：UI Implementation Agent。

5. AI 反馈阶段
   - 产出动态反馈规则、教师提示、学生即时反馈。
   - 负责人：AI Feedback Agent。

6. QA 阶段
   - 执行类型、lint、测试和构建检查。
   - 负责人：QA Agent。

7. 发布阶段
   - 更新 README、最终报告、已知问题和扩展方向。
   - 负责人：Release Agent。

## 质量门禁

- 所有阶段必须有可追踪文档。
- 所有 Agent 必须有 report。
- 物理模型必须可单元测试。
- `npm run build` 必须成功。
- 课堂页面必须真实交互，不能仅展示静态说明。
