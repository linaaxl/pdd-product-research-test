# QA Contract

## 必跑命令

发布前必须按顺序执行：

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

如果任一命令失败，必须先修复，再从失败命令开始重跑；最终报告中必须记录最后一次通过结果。

## 功能验收

- 页面包含完整的《电磁感应：磁铁穿过线圈》交互仿真。
- 磁铁支持拖拽，播放按钮支持自动运动。
- 参数包含磁铁位置、速度、线圈匝数、磁场强度、是否反向运动。
- 实时图像包含 Φ-t 曲线和 e-t 曲线。
- 至少 3 个学生互动问题。
- 至少 3 条动态反馈规则。
- 教师提示能随学生操作变化。

## 架构验收

- 物理模型在 `src/lib/` 中实现为 pure functions。
- React UI 不直接承载核心物理公式。
- 反馈规则可独立测试或独立调用。
- 不引入重型图表库或仿真库，优先 SVG/Canvas 与 TypeScript。

## 测试验收

物理模型测试至少覆盖：

- 磁铁静止时感应电动势接近 0。
- 速度增大时感应电动势幅值增大。
- 匝数增大时感应电动势幅值增大。
- 运动方向反向时感应电动势变号。

## 报告验收

- 每个 Agent 有独立 report。
- `docs/reports/final_report.md` 包含已实现功能、交互说明、教学设计、QA 命令结果、已知问题、扩展方向。
